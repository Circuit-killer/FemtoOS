58 /* DISCUSSION
59  * We don't use primitive data types in the code anywhere. All types are related
60  * to the types below. We compile with the option -mint8, otherwise gcc promotes
61  * a lot of 8 bit operations to 16 bit. Although this is standard behavior, and
62  * can be suppressed with the right casts, to my opinion this has been the wrong
63  * choice. C is a machine language and should respect the true lengths of the
64  * defined variables. Note that it seems that GCC does not honor the bit lengths
65  * defined; so why are the present anyway? With -mint8 int32_t becomes a 16 bit
66  * integer. Why? This is totally confusing.
67  */
