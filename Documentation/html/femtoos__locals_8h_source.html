32  * depend on the direction of the stack growth.
33  * Be ware, the endianness is relevant too. These values are for little endian encoding,

184 /* Definition of the task status byte
185  * 8 bit structure: smbdpppr
186  * s: stopstate (go/halt)
187  * m: mode: (suspend/sleep/access/sync)
188  * b blockstate: (blocked/free)
189  * d: delaystate
190  * ppp: priority
191  * r: dressstate (runable/done, slot/event, read,write)
192  *
193  * The first three bits can be coupled to the states as follows:
194  *
195  * 0001 terminated (suspended blocked) never delayed!
196  * 001x suspended (suspended unblocked) can be delayed [LSB kep for switchblock]
197  * 010x sleeping blocked on sync (but not on fs, this is forbidden) (LSB: slot/event) can be delayed
198  * 011x sleeping free can be delayed [LSB kept for switchblock]
199  * 100x running blocked on fs (LSB:read/write) , can be delayed, switchblock lost after release
200  * 101x running shared, can be delayed, (LSB shared runable/done), switchblock lost after release
201  * 110x running blocked (LSB: slot/event) , can be delayed, switchblock lost after release
202  * 111x running free (LSB: runable/done) , can be delayed
203  *
204  * The delay bit is orthogonal to these states, and without meaning in the
205  * terminated state. A delay in any other blocked state may end and
206  * depending on the state may wakeup the state.
207  *
208  * The defRestart States are coupled to these states.
209  */

281 /* Definition of the uiInitialStatus (start-state) byte
282  * 8 bit structure: mmsipppt
283  * mm: start-mode: (terminated, suspended, sleeping, running)
284  * s: shared-state (set/reset)
285  * i tick interrupt (set/reset)
286  * ppp: priority
287  * t: global interrupt (set/reset)
288  *
289  * The mode has the following meaning
290  * 00 start suspended
291  * 01 start sleeping
292  * 11 start running (or shared depending on share bit)
293  *
294  * To come from these states to the task status we must translate
295  * (only first nibble)
296  *
297  * 00x => 0001 (terminated)
298  * 00x => 001x (suspended)
299  * 01x => 011x (sleeping)
300  * 111 => 101x (shared)
301  * 110 => 111x (running)
302  *
303  * Converting from start-state to task-state requires the elimination
304  * of the shared task bit. That bit is the most significant bit of the
305  * start-state but is absent in the task status byte.
306  */
307

596  * we come from an idle or sleep state. Make sure this definition relates to the other
597  * defContext definitions.
598  * Note you cannot just change these definitions to you liking, assembly code has to
599  * be adapted as well.
